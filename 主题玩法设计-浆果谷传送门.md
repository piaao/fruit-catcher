## 一、核心机制：传送门系统（Portal System）

### 设计理念

浆果谷的精灵族掌握空间魔法，在关卡中开启**传送门对**——水果进入一扇门，会从另一扇门以特定方向飞出。玩家需要预判传送后的落点，同时利用传送门将水果"跳转"到角色附近。

### 视觉效果

- **传送门外观**：紫蓝色发光圆环，内有旋转的星尘粒子，像魔法传送门
- **进入特效**：水果被吸入时产生紫色漩涡
- **传出特效**：水果飞出时产生蓝色光爆+✨粒子
- **背景粒子**：萤火虫（`firefly`）+ 魔法光点

### 机制参数

```
传送门A ←→ 传送门B（成对出现，随机关方向/距离）
传送延迟：水果进入门后约 0.3-0.5 秒后从另一门飞出
飞行方向：出射方向与入射方向保持一致（不改变速度方向，只改变位置）
```

### 实现方案（参考海风系统结构）

```javascript
// ---- 传送门系统（浆果谷主题）----
let portalState = {
  active: false,
  portals: [],         // [{x, y, dx, dy, inTimer, outTimer, particles}]
  strength: 0,         // 0=无, 1=一级(2对门), 2=二级(3对门), 3=三级(4对门)
};

// 每关初始化时调用：生成传送门对
function initPortalsLevel() {
  const lvl = LEVELS[level];
  if (!lvl.portalStrength) { portalState.active = false; return; }
  portalState.active = true;
  portalState.strength = lvl.portalStrength;
  // 生成 N 对传送门，随机位置和朝向
}

// 每帧更新
function updatePortals(dt) {
  if (!portalState.active) return;
  portalState.portals.forEach(p => {
    // 门环旋转动画
    p.rotation += dt * 2;
    // 更新传送门粒子效果
  });
}

// 水果经过传送门时触发
function checkPortalTeleport(p) {
  if (!portalState.active) return p;
  for (const portal of portalState.portals) {
    const d = dist(p.x, p.y, portal.x, portal.y);
    if (d < portal.radius + p.def.radius) {
      // 找到配对门
      const pair = portalState.portals.find(p2 => p2.id === portal.pairId);
      if (pair) {
        // 从配对门传出，保持原速度方向不变
        p.x = pair.x + pair.dx * 30;
        p.y = pair.y + pair.dy * 30;
        spawnParticles(portal.x, portal.y, '#8060ff', 15);
        spawnParticles(pair.x, pair.y, '#60d0ff', 15);
      }
    }
  }
}
```

## 四、难度曲线（与通用曲线一致）

```
baseSpeed:     100 + level * 4   (184 → 220)
spawnInterval: 1800 - level * 26 (1280 → 1010ms)
bombChance:    0.06 + level * 0.0036 (0.130 → 0.170)
target:        50 + level * 16   (400 → 580)
```
